# STM32-K210 PanTilt Tracker

## 项目简介

本项目是一套低成本双轴人脸跟踪云台系统。K210 负责摄像头采集与人脸检测，计算目标中心坐标；STM32 负责接收坐标并控制两个舵机，使云台跟随目标转动。

当前版本主要实现“人脸检测 + 坐标发送 + 云台跟踪”，后续计划增加人脸识别与 LoRa 远程状态上报，扩展为一套低成本智能监控系统。

## 系统组成

```text
K210 摄像头模块
    ↓ 人脸检测 / 坐标计算
UART 串口通信
    ↓ 目标坐标数据包
STM32 云台控制器
    ↓ PWM
双轴舵机云台
```

## 当前功能

- K210 使用 YOLO2 模型进行人脸检测。
- 自动选择画面中面积最大的人脸作为跟踪目标。
- 计算人脸中心点坐标 `(x, y)`。
- 通过 UART 向 STM32 发送 7 字节数据包。
- STM32 解析坐标后控制水平、俯仰两个舵机。
- 支持 PID 版本和非 PID 版本。
- 支持目标死区、串口解析修复、错误状态显示优化。
- 无目标时发送 `0xFFFF, 0xFFFF`，通知下位机停止跟踪。

## 串口协议

```text
0xAA 0x07 XH XL YH YL 0x88
```

| 字节 | 含义 |
|---|---|
| `0xAA` | 包头 1 |
| `0x07` | 包头 2 / 数据长度标识 |
| `XH XL` | 目标中心 X 坐标，高字节在前 |
| `YH YL` | 目标中心 Y 坐标，高字节在前 |
| `0x88` | 包尾 |

无目标指令：

```text
X = 65535
Y = 65535
```

## 算法说明

### 1. 人脸检测

K210 采集 320×240 图像，并使用 KPU 加速 YOLO2 人脸检测模型。检测到多个人脸时，程序选择面积最大的人脸作为目标。

### 2. 坐标计算

检测框坐标为 `(x, y, w, h)`，目标中心点计算方式为：

```text
face_x = x + w / 2
face_y = y + h / 2
```

### 3. 云台控制

STM32 将目标坐标与画面中心点 `(160, 120)` 比较，得到像素误差，再换算为舵机角度修正量。

非 PID 版本直接根据角度偏差修正舵机位置；PID 版本在误差基础上加入比例、积分、微分控制，使跟踪过程更平滑。

### 4. 死区处理

当目标接近画面中心时，小范围误差不会立即驱动舵机，避免舵机因检测抖动频繁来回摆动。

## 后续计划

### 1. 增加人脸识别

当前系统只能检测“有没有人脸”，不能判断“是谁”。后续计划在 K210 或其他边缘 AI 模块上加入人脸特征提取与比对功能，实现：

- 检测到人脸；
- 判断是否为授权人员；
- 输出“正确人脸 / 非授权人脸”状态；
- 将识别结果发送给 STM32 或通信模块。

### 2. 增加 LoRa 远程上报

后续计划加入 LoRa 模块，用于远距离、低功耗状态通信。系统不传输视频，只发送少量状态信息，例如：

```text
FACE_OK
FACE_UNKNOWN
NO_TARGET
DEVICE_ERROR
```

这样可以组成一套低成本、低功耗、远距离的基础监控系统。

### 3. 系统目标

最终系统目标：

```text
人脸检测
  + 人脸识别
  + 云台自动跟踪
  + LoRa 远程状态上报
  = 低成本智能监控系统
```

## 注意事项

- 当前代码是嵌入式实验项目，不是成熟安防产品。
- 人脸识别和监控功能应遵守当地法律法规，并在获得授权和知情同意的情况下使用。
- LoRa 适合传输低频状态数据，不适合传输图像或视频。
- 如需更高识别率，可更换更高质量的人脸检测 / 人脸识别模型。
